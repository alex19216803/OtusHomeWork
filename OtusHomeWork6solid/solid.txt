1. S - Single Responsibility Principle (Принцип единственной ответственности)
GameSettings - отвечает только за хранение настроек игры

RandomNumberGenerator - отвечает только за генерацию случайных чисел

ConsoleUI - отвечает только за взаимодействие с пользователем через консоль

GameEngine - отвечает только за логику игры (бизнес-логику)

ConsoleLogger/FileLogger - отвечают только за логирование

2. O - Open/Closed Principle (Принцип открытости/закрытости)
Классы открыты для расширения, но закрыты для модификации:

Можно добавить новый генератор чисел (например, FixedNumberGenerator для тестов), реализовав INumberGenerator

Можно добавить новый UI (например, WindowsFormsUI или WebAPIUI), реализовав IGameUI

Можно добавить новый способ логирования (например, DatabaseLogger), реализовав IGameLogger

3. L - Liskov Substitution Principle (Принцип подстановки Барбары Лисков)
Все реализации интерфейсов могут быть взаимозаменяемы:

ConsoleLogger и FileLogger можно использовать вместо друг друга без изменения логики игры

Любая реализация INumberGenerator корректно работает в GameEngine

Любая реализация IGameUI может быть подставлена в игру

4. I - Interface Segregation Principle (Принцип разделения интерфейсов)
Интерфейсы разделены по функциональному назначению:

IGameSettings - только настройки

INumberGenerator - только генерация чисел

IGameLogger - только логирование

IGameUI - только пользовательский интерфейс

Нет "жирных" интерфейсов, которые содержат неиспользуемые методы

5. D - Dependency Inversion Principle (Принцип инверсии зависимостей)
Модули верхнего уровня (GameEngine) не зависят от модулей нижнего уровня (конкретных реализаций)
